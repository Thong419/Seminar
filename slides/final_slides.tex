\begin{frame}{Problem Statement}
  \begin{itemize}
    \item Fake news spreads quickly and overwhelms manual verification workflows.
    \item Binary classification alone is not enough for reviewer trust.
    \item A practical system must combine accuracy, explanation, evidence, and deployability.
  \end{itemize}
  \note{Speaker notes: Emphasize operational pain: speed, scale, and reviewer burden. Defense talking points: realistic business framing, moderation support. Expected questions: Why is this a meaningful NLP problem?}
\end{frame}

\begin{frame}{Objectives}
  \begin{itemize}
    \item Build a reproducible baseline and a stronger transformer model.
    \item Deploy the model through FastAPI and Streamlit.
    \item Add an agentic layer for evidence retrieval, decision synthesis, and uncertainty handling.
    \item Address monitoring, privacy, robustness, and ethics.
  \end{itemize}
  \note{Speaker notes: This slide maps directly to rubric requirements. Defense talking points: end-to-end design, not just modeling. Expected questions: Which objectives were most important for grading?}
\end{frame}

\begin{frame}{Dataset}
  \begin{columns}[T,onlytextwidth]
    \column{0.52\textwidth}
    \begin{itemize}
      \item Local FakeNewsNet-style CSV configured in \texttt{configs/dataset.yaml}.
      \item Fields: \texttt{text}, \texttt{label}; labels: fake and real.
      \item Stratified split: 70\% train, 15\% validation, 15\% test.
    \end{itemize}
    \column{0.44\textwidth}
    \begin{block}{Preprocessing}
      \begin{itemize}
        \item Remove HTML and URLs
        \item Lowercase text
        \item Normalize whitespace
        \item Remove special characters
      \end{itemize}
    \end{block}
  \end{columns}
  \note{Speaker notes: Mention that preprocessing and splitting live in code, not only in notebooks. Defense talking points: reproducibility, stratification, data handling. Expected questions: How do you prevent leakage or class imbalance issues?}
\end{frame}

\begin{frame}{RoBERTa Model}
  \begin{columns}[T,onlytextwidth]
    \column{0.46\textwidth}

      \item Max length 512, batch size 16, learning rate $2\times10^{-5}$, 3 epochs.
      \item Best checkpoint selected by validation F1.

\begin{frame}{Evidence Retrieval}
  \begin{itemize}
    \item Hybrid HTML-based retrieval improves demo robustness without requiring a single paid API.
    \item Sources are scored by relevance, credibility, stance, and matched terms.
    \item Aggregated outputs: support score, contradiction score, quality score, conflict flag.
  \end{itemize}
  \note{Speaker notes: Explain the practical fallback strategy and structured evidence bundle. Defense talking points: source credibility, public web dependency, heuristic stance detection. Expected questions: Why evidence retrieval? Why not rely only on confidence?}
\end{frame}

\begin{frame}{Explainability}
  \begin{itemize}
    \item Final explanation includes prediction, token importance, evidence summary, and review state.
    \item Trust score comes from the decision layer and stays consistent in the UI.
    \item UNCERTAIN is surfaced when confidence or evidence is insufficient.
  \end{itemize}
  \note{Speaker notes: Clarify difference between confidence and trust score. Defense talking points: consistent trust value, user-facing reasoning, uncertainty handling. Expected questions: Why do you need both confidence and trust?}
\end{frame}

\begin{frame}{Evaluation}
  \begin{itemize}
    \item Offline metrics: accuracy, precision, recall, F1, confusion matrix.
    \item Online-style checks: API tests, agent endpoint tests, trace serialization, monitoring writes.
    \item Comparison is always against the TF-IDF baseline.
  \end{itemize}
  \note{Speaker notes: Separate model evaluation from system evaluation. Defense talking points: held-out testing, endpoint validation, orchestration behavior. Expected questions: How did you validate the agent workflow itself?}

\begin{frame}{Demo Flow}
  \begin{enumerate}
    \item User submits article text in Streamlit.
    \item FastAPI masks PII and routes the request.
    \item Agent classifies, retrieves evidence, makes a decision, and generates explanation.
    \item UI shows prediction, trust, evidence cards, and trace timeline.
  \end{enumerate}
  \note{Speaker notes: This is the live demo script condensed into one slide. Defense talking points: explain what the examiner will see step by step. Expected questions: Which endpoint powers the final frontend?}
\end{frame}

\begin{frame}{Results}
  \begin{columns}[T,onlytextwidth]
    \column{0.56\textwidth}
    \begin{itemize}
      \item RoBERTa outperforms the baseline on all core metrics.
      \item Agent layer adds evidence, conflict detection, and explainability without retraining the checkpoint.
      \item Monitoring and traces improve defense readiness and operational visibility.
    \end{itemize}

\begin{frame}{Monitoring and Lifecycle}
  \begin{itemize}
    \item Monitoring modules track confidence drift, prediction drift, and model health.
    \item Agent metrics capture evidence success rate, conflict rate, uncertain rate, and response time.
    \item Retraining strategy is designed conceptually and partially implemented through monitoring outputs.
  \end{itemize}
  \note{Speaker notes: Link this directly to the continual learning requirement. Defense talking points: concept vs full automation, alert-driven retraining, file-based artifacts. Expected questions: Why is retraining not fully automated?}
\end{frame}

\begin{frame}{Limitations}
  \begin{itemize}
    \item Evidence retrieval depends on public web availability.
    \item Evidence stance analysis is heuristic, not learned.
    \item Public production deployment still needs stronger observability and rate limiting.
  \end{itemize}
  \note{Speaker notes: Be explicit and non-defensive. Defense talking points: realistic scope control, no hidden weaknesses. Expected questions: What would fail first in a public deployment?}
\end{frame}

\begin{frame}{Future Work}
  \begin{itemize}

    \item Expand robustness and fairness evaluation across broader domains.
    \item Add managed observability, CI-based release flow, and reviewer feedback loop.
  \end{itemize}
  \note{Speaker notes: Present future work as natural next steps, not admissions of failure. Defense talking points: retrieval quality, cloud hardening, human-in-the-loop learning. Expected questions: What is the highest-ROI next improvement?}
\end{frame}

  \end{itemize}
  \note{Speaker notes: Close on completeness and coherence. Defense talking points: end-to-end, explainable, deployable, monitored. Expected questions: What single claim best summarizes the project?}
